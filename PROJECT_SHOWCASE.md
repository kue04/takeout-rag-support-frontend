# 外卖客服 RAG 前端工作台项目展示

## 项目定位

这是外卖客服 RAG 系统的 React 前端工作台。它把用户下单、订单售后、客服对话、RAG 诊断、反馈采集和知识运营整合在一个可演示界面里，用来说明 AI 客服系统如何从 Demo 走向可运营应用。

## 主要模块

- 外卖业务模拟：门店、购物车、订单和售后入口。
- 客服对话：围绕订单问题发起 RAG 问答。
- RAG 诊断：通过 5 个 tab 展示流程时间线、工具调用、证据溯源、上下文记忆和原始 JSON。
- 回答依据：在客服回复下方展示主证据、引用片段、订单工具摘要、规则兜底和人工接管原因。
- 订单场景演示：支持快速切换未接单、商家制作、骑手取餐、已送达等状态，演示订单状态如何影响客服回复。
- 反馈闭环：提交有帮助/无帮助反馈，导出 eval case。
- 知识运营：新增、修改、审核、发布和回滚知识条目。
- 项目展示：面向面试的系统介绍页。

## 技术栈

- React 19
- TypeScript
- Vite
- Tailwind CSS
- GSAP
- lucide-react

## 面试亮点

1. 前端不只是聊天 UI，而是把 RAG 链路调试能力产品化。
2. 用户侧只看到客服回复，演示侧能看到 full trace、tool results、evidence citations 和 memory snapshot。
3. 通过预置订单场景展示工具调用和业务状态如何约束模型回复，避免“模型凭空承诺退款”。
4. 通过知识运营页面展示“知识变更如何进入 RAG”的真实工作流。
5. 与后端评测、反馈、诊断接口联动，体现 AI 应用闭环意识。

## 当前前端增强

- `/chat/prompt` 请求已支持 `user_id`、`session_id`、`order_id`，并保留旧字段兼容。
- 诊断面板优先读取 `full_trace`，没有该字段时回退展示旧 `trace`。
- 证据页优先读取 `evidence_citations`，没有该字段时回退展示检索结果。
- 记忆页展示短期摘要、订单状态、长期记忆字段和运行指标。
- 原始 JSON 页完整展示后端响应，便于答辩时解释 RAG 工作流。

